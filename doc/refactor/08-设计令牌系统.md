# 08 - 设计令牌系统

## 🎯 目标

建立统一的设计令牌系统，减少硬编码的样式值，提升设计一致性和可维护性。

## 📊 当前问题

### 硬编码的设计值

**发现的问题**：
- 间距值分散（`space-y-6`, `space-y-3`, `gap-4` 等）
- 图标尺寸不统一（`h-4 w-4`, `h-5 w-5` 等）
- 排版样式重复（`text-3xl font-bold` 等）
- 缺少语义化的设计常量

**影响**：
- ❌ 设计不一致
- ❌ 修改困难（需要全局搜索替换）
- ❌ 新开发者不知道该用哪个值
- ❌ 难以实现设计系统

---

## 🎨 设计令牌系统设计

### 令牌分类

```
设计令牌
├── 间距令牌（Spacing Tokens）
│   ├── 容器间距
│   ├── 区块间距
│   └── 内联间距
├── 排版令牌（Typography Tokens）
│   ├── 标题样式
│   ├── 正文样式
│   └── 辅助文本样式
├── 布局令牌（Layout Tokens）
│   ├── 页面布局
│   ├── Flex 布局
│   └── Grid 布局
├── 交互令牌（Interactive Tokens）
│   ├── 按钮样式
│   ├── 链接样式
│   └── 状态样式
└── 尺寸令牌（Size Tokens）
    ├── 图标尺寸
    ├── 头像尺寸
    └── 容器尺寸
```

// ============================================
// 外部链接
// ============================================

export const EXTERNAL_LINKS = {
  GITHUB: 'https://github.com',
  TWITTER: 'https://twitter.com',
  DOCUMENTATION: 'https://docs.example.com',
} as const

/**
 * 构建带查询参数的 URL
 */
export function buildUrl(path: string, params?: Record<string, string | number>): string {
  if (!params) return path

  const searchParams = new URLSearchParams()
  Object.entries(params).forEach(([key, value]) => {
    searchParams.append(key, String(value))
  })

  return `${path}?${searchParams.toString()}`
}

/**
 * 构建分页 URL
 */
export function buildPaginationUrl(basePath: string, page: number, otherParams?: Record<string, string>): string {
  return buildUrl(basePath, { page, ...otherParams })
}

## 🎯 最佳实践

### 1. 优先使用令牌

```tsx
// ✅ 好的做法
<div className={spacing.section}>

// ❌ 不好的做法
<div className="space-y-6">
```

### 2. 组合令牌

```tsx
// ✅ 好的做法
import { spacing, typography } from '@/lib/constants'
<div className={cn(spacing.container, typography.body)}>

// ❌ 不好的做法
<div className="container mx-auto py-8 px-4 text-base">
```

### 3. 特殊情况可以混用

```tsx
// ✅ 允许：令牌 + 特殊样式
<div className={cn(layout.flexBetween, "bg-red-500")}>
```

### 4. 保持令牌语义化

```tsx
// ✅ 语义清晰
spacing.section
typography.h1

// ❌ 语义不清
spacing.s6
typography.t1
```

# 05 - UI 组件抽象

## 🎯 目标

提取重复的 UI 模式，建立共享组件库，统一视觉体验，减少代码重复。

## 📊 当前问题

### 重复的 UI 模式统计

| UI 模式 | 出现次数 | 代码量 | 位置 |
|---------|---------|--------|------|
| 空状态提示 | 3 处 | ~15 行/处 | blog、tags、admin |
| 错误状态展示 | 4 处 | ~20 行/处 | blog、comments、admin |
| 加载状态 | 5 处 | ~10 行/处 | 各页面 |
| 骨架屏 | 分散 | ~150 行 | blog、comments |
| 页面布局 | 5+ 处 | ~30 行/处 | 各页面 |
| 页面头部 | 4 处 | ~15 行/处 | blog、admin |

**总计**：约 400 行重复代码

### 步骤 1：创建空状态组件

```typescript
// components/shared/empty-state.tsx - 新建

import { Card, CardContent } from '@/components/ui/card'
import { Button } from '@/components/ui/button'
import type { LucideIcon } from 'lucide-react'
import { ReactNode } from 'react'

export interface EmptyStateProps {
  /**
   * 图标组件
   */
  icon?: LucideIcon
  /**
   * 标题
   */
  title: string
  /**
   * 描述文本
   */
  description?: string
  /**
   * 操作按钮或自定义内容
   */
  action?: ReactNode
  /**
   * 自定义类名
   */
  className?: string
}

/**
 * 空状态组件
 * 用于展示无数据时的提示
 */
export function EmptyState({
  icon: Icon,
  title,
  description,
  action,
  className,
}: EmptyStateProps) {
  return (
    <Card className={className}>
      <CardContent className="py-12 text-center">
        {Icon && (
          <Icon className="mx-auto h-12 w-12 text-muted-foreground mb-4" />
        )}
        <h3 className="text-lg font-semibold mb-2">{title}</h3>
        {description && (
          <p className="text-muted-foreground mb-4">{description}</p>
        )}
        {action}
      </CardContent>
    </Card>
  )
}
```

#### 使用示例

```tsx
// 改造前
{posts.length === 0 && (
  <Card>
    <CardContent className="py-12 text-center text-muted-foreground">
      暂无文章
    </CardContent>
  </Card>
)}

// 改造后
import { EmptyState } from '@/components/shared/empty-state'
import { FileText } from 'lucide-react'

{posts.length === 0 && (
  <EmptyState
    icon={FileText}
    title="暂无文章"
    description="还没有发布任何文章"
    action={
      <Button asChild>
        <Link href="/write">开始创作</Link>
      </Button>
    }
  />
)}
```

```typescript
// components/shared/error-state.tsx - 新建

import { Card, CardContent } from '@/components/ui/card'
import { Button } from '@/components/ui/button'
import { AlertCircle, RefreshCw } from 'lucide-react'

export interface ErrorStateProps {
  /**
   * 错误信息
   */
  error: string
  /**
   * 重试回调
   */
  onRetry?: () => void
  /**
   * 自定义类名
   */
  className?: string
}

/**
 * 错误状态组件
 * 用于展示错误信息和重试按钮
 */
export function ErrorState({ error, onRetry, className }: ErrorStateProps) {
  return (
    <Card className={className}>
      <CardContent className="py-12 text-center">
        <AlertCircle className="mx-auto h-12 w-12 text-destructive mb-4" />
        <h3 className="text-lg font-semibold mb-2">出错了</h3>
        <p className="text-muted-foreground mb-4">{error}</p>
        {onRetry && (
          <Button onClick={onRetry} variant="outline">
            <RefreshCw className="mr-2 h-4 w-4" />
            重试
          </Button>
        )}
      </CardContent>
    </Card>
  )
}
```

#### 使用示例


```typescript
// components/shared/loading-state.tsx - 新建

import { Loader2 } from 'lucide-react'
import { cn } from '@/lib/utils'

export interface LoadingStateProps {
  /**
   * 加载提示文本
   */
  message?: string
  /**
   * 尺寸
   */
  size?: 'sm' | 'md' | 'lg'
  /**
   * 自定义类名
   */
  className?: string
}

const sizeMap = {
  sm: 'h-4 w-4',
  md: 'h-8 w-8',
  lg: 'h-12 w-12',
}

/**
 * 加载状态组件
 * 用于展示加载中的提示
 */
export function LoadingState({
  message = '加载中...',
  size = 'md',
  className,
}: LoadingStateProps) {
  return (
    <div className={cn('flex items-center justify-center py-12', className)}>
      <Loader2
        className={cn(
          'animate-spin text-muted-foreground mr-3',
          sizeMap[size]
        )}
      />
      <span className="text-muted-foreground">{message}</span>
    </div>
  )
}
```

### 步骤 4：创建页面布局组件

```typescript
// components/shared/page-layout.tsx - 新建

import { ReactNode } from 'react'
import { cn } from '@/lib/utils'

export interface PageLayoutProps {
  /**
   * 页面标题
   */
  title?: string
  /**
   * 页面描述
   */
  description?: string
  /**
   * 头部操作按钮
   */
  actions?: ReactNode
  /**
   * 侧边栏内容
   */
  sidebar?: ReactNode
  /**
   * 主内容
   */
  children: ReactNode
  /**
   * 自定义类名
   */
  className?: string
}

/**
 * 页面布局组件
 * 提供统一的页面布局结构（主内容 + 侧边栏）
 */
export function PageLayout({
  title,
  description,
  actions,
  sidebar,
  children,
  className,
}: PageLayoutProps) {
  return (
    <div className={cn('container mx-auto py-8 px-4', className)}>
      <div className="grid grid-cols-1 lg:grid-cols-4 gap-8">
        <div className="lg:col-span-3 space-y-6">
          {(title || description || actions) && (
            <PageHeader
              title={title}
              description={description}
              actions={actions}
            />
          )}
          {children}
        </div>
        {sidebar && <aside className="space-y-6">{sidebar}</aside>}
      </div>
    </div>
  )
}

/**
 * 页面头部组件
 */
interface PageHeaderProps {
  title?: string
  description?: string
  actions?: ReactNode
}

function PageHeader({ title, description, actions }: PageHeaderProps) {
  return (
    <div className="flex items-center justify-between">
      <div>
        {title && <h1 className="text-3xl font-bold">{title}</h1>}
        {description && (
          <p className="text-muted-foreground mt-2">{description}</p>
        )}
      </div>
      {actions && <div className="flex items-center gap-2">{actions}</div>}
    </div>
  )
}
```

## ⚠️ 注意事项

### 1. 避免过度抽象

**不要抽取的组件**：
- ❌ 只使用 1-2 次的 UI
- ❌ 业务特定的复杂组件
- ❌ 需要大量 props 配置的组件

**应该抽取的组件**：
- ✅ 使用 3+ 次的 UI 模式
- ✅ 语义明确的通用组件
- ✅ 视觉一致性要求高的组件

### 2. 保持灵活性

```tsx
// ✅ 好的设计：支持自定义
<EmptyState
  icon={FileText}
  title="暂无文章"
  action={<CustomButton />}  // 允许自定义
/>

// ❌ 不好的设计：过度封装
<EmptyState type="no-posts" />  // 失去灵活性
```

### 3. 类型安全

所有共享组件都应该有完整的 TypeScript 类型定义。

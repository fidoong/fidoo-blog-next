# 02 - 类型系统重构

## 🎯 目标

消除项目中的 21 处类型断言，建立精确的类型系统，提升类型安全性。

## 📊 当前问题

### 类型断言统计

| 文件 | 断言次数 | 典型案例 |
|------|---------|---------|
| `app/blog/[slug]/page.tsx` | 8 处 | `post.author as { name?: string }` |
| `app/(admin)/posts/posts-content.tsx` | 3 处 | `post.category as { name?: string }` |
| `components/blog/post-card.tsx` | 4 处 | `post.author as PostAuthor` |
| `components/blog/comment-item.tsx` | 3 处 | `comment.author as { ... }` |
| 其他文件 | 3 处 | 各种关系类型断言 |
| **总计** | **21 处** | - |

### 问题根源

```typescript
// 当前的类型定义（types/models.ts）
export interface Post extends TimestampFields {
  id: string
  slug: string
  title: string
  // ...
  authorId: string
  author: PublicUser  // ❌ 但实际查询返回的结构不同
  categoryId: string
  category: Category  // ❌ 但实际查询返回的结构不同
  tags: Tag[]         // ❌ 但实际是 { tag: Tag }[] 结构
}
```

**问题**：
1. 类型定义与数据库查询返回的实际结构不匹配
2. Drizzle ORM 的 `with` 查询返回嵌套结构
3. 缺少针对不同场景的精确类型

## 🔧 解决方案

### 方案设计

创建**场景化的精确类型**，而不是一个通用的 `Post` 类型。

```
Post (基础类型)
  ├── PostWithRelations (详情页、编辑页)
  ├── PostSummary (列表页)
  ├── AdminPostListItem (管理后台)
  └── PostFormData (表单数据)
```

#### 1.2 定义类型转换工具

```typescript
// types/transformers.ts - 新建

import type { PostWithRelations, PostSummary, CommentWithAuthor } from './models'

/**
 * 转换 Drizzle 查询结果为 PostWithRelations
 */
export function transformPostWithRelations(raw: any): PostWithRelations {
  return {
    ...raw,
    author: {
      id: raw.author.id,
      username: raw.author.username,
      name: raw.author.name,
      avatar: raw.author.avatar,
      bio: raw.author.bio,
    },
    category: {
      id: raw.category.id,
      name: raw.category.name,
      slug: raw.category.slug,
      description: raw.category.description,
    },
    tags: raw.tags.map((pt: any) => pt.tag),
  }
}

/**
 * 转换为 PostSummary
 */
export function transformPostSummary(raw: any): PostSummary {
  return {
    id: raw.id,
    slug: raw.slug,
    title: raw.title,
    excerpt: raw.excerpt,
    coverImage: raw.coverImage,
    published: raw.published,
    featured: raw.featured,
    views: raw.views,
    likesCount: raw.likesCount,
    commentsCount: raw.commentsCount,
    readingTime: raw.readingTime,
    createdAt: raw.createdAt,
    publishedAt: raw.publishedAt,
    author: {
      id: raw.author.id,
      username: raw.author.username,
      name: raw.author.name,
      avatar: raw.author.avatar,
      bio: raw.author.bio,
    },
    category: {
      id: raw.category.id,
      name: raw.category.name,
      slug: raw.category.slug,
      description: raw.category.description,
    },
    tags: raw.tags.map((pt: any) => pt.tag),
  }
}

/**
 * 转换评论为 CommentWithAuthor
 */
export function transformComment(raw: any): CommentWithAuthor {
  return {
    ...raw,
    author: {
      id: raw.author.id,
      username: raw.author.username,
      name: raw.author.name,
      avatar: raw.author.avatar,
      bio: raw.author.bio,
    },
    replies: raw.replies?.map(transformComment) ?? [],
  }
}
```

### 步骤 2：修改数据查询代码

#### 2.1 修改 `lib/actions/posts.ts`

```typescript
// lib/actions/posts.ts

import { transformPostWithRelations, transformPostSummary } from '@/types/transformers'
import type { PostWithRelations, PostSummary, AdminPostListItem } from '@/types/models'

// 获取文章详情
export async function getPostBySlug(slug: string): Promise<PostWithRelations> {
  const raw = await db.query.posts.findFirst({
    where: eq(posts.slug, slug),
    with: {
      author: {
        columns: {
          id: true,
          username: true,
          name: true,
          avatar: true,
          bio: true,
        },
      },
      category: true,
      tags: {
        with: {
          tag: true,
        },
      },
    },
  })

  if (!raw) {
    throw new Error('Post not found')
  }

  // ✅ 使用转换函数，返回精确类型
  return transformPostWithRelations(raw)
}

// 获取文章列表
export async function getPosts(params: GetPostsParams): Promise<PaginatedResult<PostSummary>> {
  const postsList = await db.query.posts.findMany({
    // ... 查询逻辑
    with: {
      author: {
        columns: { id: true, username: true, name: true, avatar: true, bio: true },
      },
      category: true,
      tags: { with: { tag: true } },
    },
  })

  // ✅ 转换为 PostSummary 类型
  return {
    data: postsList.map(transformPostSummary),
    total: totalCount,
    page: params.page,
    pageSize: params.limit,
    totalPages: Math.ceil(totalCount / params.limit),
    hasMore: params.page * params.limit < totalCount,
  }
}
```

## 📝 注意事项

### 1. 渐进式迁移

如果一次性修改风险太大，可以分步进行：

**第一步**：只修改类型定义和转换函数  
**第二步**：修改 Server Actions  
**第三步**：修改组件文件  

### 2. 保持向后兼容

在过渡期，可以同时保留旧类型：

```typescript
// 标记为废弃
/** @deprecated 使用 PostWithRelations 替代 */
export type PostWithRelations_Old = Post
```

### 3. 类型导出

确保所有新类型都从 `types/index.ts` 导出：

```typescript
// types/index.ts
export * from './models'
export * from './transformers'
export * from './forms'
export * from './api'
```
